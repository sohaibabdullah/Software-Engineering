About This Course

Welcome to CSE 3225 Software Engineering. This course goes beyond writing code. It teaches you how to think, plan, and collaborate like a professional software engineer — skills that remain essential even as AI tools transform how software is built.

Modern software systems power everything from hospital equipment to financial infrastructure to social media platforms. Building them correctly, on time, and within budget is one of the hardest challenges in engineering. This course equips you with the systematic methods, tools, and habits of mind needed to succeed at that challenge.

🎯 What This Course is About

Software engineering is defined as the systematic, disciplined, and quantifiable approach to developing, operating, maintaining, and evolving software systems. It is far more than programming — it encompasses requirements, design, testing, teamwork, and professional responsibility.

In this course you will learn to work the way industry teams work: writing requirements as user stories, building in short iterations, testing before coding, and using AI tools responsibly rather than blindly.

Why Software Engineering is Hard

Frederick Brooks identified four essential difficulties that make software uniquely challenging and that no technology — including AI — can fully eliminate:

  • Complexity — software systems are among the most intricate constructions ever built by humans
  • Conformity — software must continuously adapt to changing laws, markets, and organizations
  • Changeability — successful software always attracts requests for more features
  • Invisibility — you cannot see software's structure the way you can see a bridge or a circuit

Survey data consistently shows that fewer than 16% of software projects are completed on time and within budget. This course teaches the methods — Agile, BDD, TDD — that give teams the best chance of being in that 16%.

🤖 Software Engineering in the Age of AI

AI coding assistants now generate 70–90% of new code in many organizations. This does not make software engineering easier or less important — it changes where the skill is required. Engineering quality migrates from writing code to specifying, testing, and reviewing it.

As Chad Fowler wrote: "You have to know what you actually want. You have to be able to recognize when you've gotten it." This course trains precisely those skills.
